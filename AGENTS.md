# Kochfeinde ‚Äî Agent Instructions

A full-stack recipe management web app (Go + PostgreSQL backend, React 19 + TypeScript frontend).

## Quick Start

```bash
just dev          # Run all: air (Go hot-reload) + vite + caddy (reverse proxy)
just build        # Build Go backend
just sqlc         # Regenerate SQL repository code after changing files in backend/queries/
just oapi         # Bundle OpenAPI spec + regenerate Go API code
just migrate-reset # Drop/recreate DB, run migrations
npm run openapi   # (in frontend/) Regenerate TS types from spec ‚Üí src/api/scheme.ts
```

**Dev URL**: `http://localhost:8000` ‚Äî Caddy proxies `/api/*` ‚Üí Go (`:8080`), everything else ‚Üí Vite (`:5173`).

## Architecture

### Backend (`backend/`)

| Layer | Location | Pattern |
|---|---|---|
| SQL queries | `backend/queries/` | Raw SQL files, sqlc-annotated (`--name: :one` / `:many`) |
| Repository | `backend/internal/repository/` | **Generated** by sqlc ‚Äî do not edit manually. Run `just sqlc` |
| API spec | `backend/spec/spec.yaml` | OpenAPI 3.0.3 ‚Äî edit this, then `just oapi` to regenerate |
| API gen | `backend/internal/api/gen.go` | **Generated** by oapi-codegen ‚Äî do not edit manually |
| API impl | `backend/internal/impl/` | Implement the `ServerInterface` methods here |
| Init/Spec | `backend/internal/spec/init.go` | Middleware chain setup, spec embedding, handler wiring |
| Middleware | `backend/internal/middleware/logger.go` | Logging middleware |
| DB lib | `backend/internal/lib/database.go` | pgxpool setup from `DATABASE_URL` env var |
| Entry | `backend/cmd/main.go` | Server bootstrap, CORS, middleware stack |
| Migrations | `backend/migrations/` | PostgreSQL migrations (golang-migrate format) |

**Data flow**: `HTTP ‚Üí CORS ‚Üí Logging ‚Üí OpenAPI validation ‚Üí Strict handler ‚Üí ServerImpl`

### Frontend (`frontend/`)

| Layer | Location | Pattern |
|---|---|---|
| Routes | `frontend/src/routes/` | File-based routing via TanStack Router |
| Root layout | `frontend/src/routes/__root.tsx` | `<Header /> + <Outlet />` layout |
| Components | `frontend/src/components/` | React components, functional components |
| Styling | Tailwind CSS v4 + daisyUI v5 | `@import "tailwindcss"` + `@plugin "daisyui"` in `index.css` |
| API client | `frontend/src/api/queries.ts` | `openapi-fetch` client with auto-refresh on 401 |
| API types | `frontend/src/api/scheme.ts` | **Generated** TS types from OpenAPI spec ‚Äî do not edit manually. Run `npm run openapi` |
| Query wrappers | `frontend/src/api/queries/` | React Query `queryOptions` builders per resource (e.g. `recipe.ts`) |
| Data fetching | TanStack React Query | `staleTime: 5000` in query client |
| Entry | `frontend/src/main.tsx` | Router + QueryClient provider setup |

**Route tree** is auto-generated by `@tanstack/router-plugin` ‚Äî restart Vite after adding routes.

### Database

- PostgreSQL, `pgx/v5` driver, `pgxpool` connection pooling
- UUID primary keys with `uuidv7()` (time-ordered)
- Foreign keys with `ON DELETE CASCADE`
- Nullable fields use `pgtype.Text`, `pgtype.UUID`, etc.
- Tables: `users`, `ingredients`, `recipes`, `recipe_step`, `recipe_ingredients`

### Conventions

- **Go**: Standard library `net/http`, oapi-codegen strict-server pattern. Functions return `(result, error)`. Packages are lowercase single words.
- **TypeScript**: `noUnusedLocals`/`noUnusedParameters` strict ‚Äî clean up unused code. `erasableSyntaxOnly: true` ‚Äî no enums, no namespaces. `jsx: "react-jsx"`.
- **Generated code**: Never edit `internal/repository/*.go`, `internal/api/gen.go`, `internal/spec/spec.gen.yaml`, `src/routeTree.gen.ts`, or `src/api/scheme.ts` manually. Regenerate with `just sqlc`, `just oapi`, `npm run openapi`, or restart Vite respectively.
- **CSS**: Tailwind utility classes in JSX. daisyUI components + themes (`light`/`dark`). Edit themes in `index.css`.
- **API**: REST under `/api/v1`. Edit `backend/spec/` first, then regenerate.
- **Testing**: None exist yet ‚Äî create as needed.
